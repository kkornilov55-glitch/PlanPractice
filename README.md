# Система управления базой данных производственного предприятия

Программный комплекс для автоматизации учета производимой продукции, расхода сырья и энергоресурсов на предприятии. Приложение реализует многопользовательский доступ к данным, выполнение CRUD-операций и формирование аналитических отчетов.

## Технические характеристики

* Язык программирования: C# (версия 13.0)
* Платформа разработки: .NET 9.0
* Технология графического интерфейса: WPF (Windows Presentation Foundation)
* Технология доступа к данным: ADO.NET
* СУБД: Microsoft Access (.accdb)
* Среда тестирования: MSTest

## Системные требования и зависимости

Проект разработан и протестирован на операционной системе **Windows 11**.

Для корректного запуска и работы приложения на компьютере пользователя должны быть установлены следующие компоненты:

1. **.NET Desktop Runtime (версия 6.0 или новее)**
Обязательная среда выполнения для работы оконных приложений, написанных на WPF.
Ссылка на скачивание: [Официальный сайт Microsoft .NET](https://dotnet.microsoft.com/ru-ru/download/dotnet)
2. **Microsoft Access Database Engine 2016 Redistributable**
Необходимый драйвер OLE DB (`Microsoft.ACE.OLEDB.12.0`). Без него приложение не сможет подключиться к файлу базы данных `.accdb`, если на компьютере не установлен Microsoft Office.
Ссылка на скачивание: [Центр загрузки Microsoft](https://www.microsoft.com/en-us/download/details.aspx?id=54920)
*Примечание: Устанавливаемая разрядность драйвера (x86 или x64) должна строго совпадать с разрядностью компиляции данного приложения.*

## Структура проекта

Решение разделено на три независимых модуля:

* **PlanPractice.UI** — исполняемый проект графического интерфейса (XAML-разметка окон и привязка данных).
* **PlanPractice.Logic** — библиотека классов. Содержит ядро приложения (`DataBaseManager`), отвечающее за прямую работу с базой данных, выполнение SQL-запросов и алгоритмы хеширования паролей по стандарту SHA-256.
* **PlanPractice.Tests** — проект автоматизированного интеграционного тестирования, покрывающий основную бизнес-логику и безопасность.

## Инструкция по запуску

1. Скачайте архив с проектом и распакуйте его в удобную директорию.
2. Убедитесь, что в системе установлены библиотеки из раздела "Системные требования".
3. Файл базы данных (`Plan.accdb`) должен находиться в одной директории с исполняемым файлом программы (`PlanPractice.UI.exe`), либо путь к нему должен быть явно указан в константе `ConnectionString` класса `DataBaseManager`.
4. Запустите исполняемый файл `PlanPractice.UI.exe`.

## Основные функциональные возможности

* **Система авторизации:** Ролевая модель доступа (Администратор, Менеджер, Гость), ограничивающая видимость системных таблиц и доступ к операциям редактирования.
* **Управление данными:** Возможность просмотра, добавления, редактирования и безопасного удаления записей с автоматической поддержкой ссылочной целостности на уровне БД.
* **Аналитика:** Выполнение встроенных и пользовательских SQL-запросов через отдельное окно.
* **Экспорт:** Выгрузка результатов SQL-выборок в текстовый формат CSV для интеграции с табличными процессорами.
